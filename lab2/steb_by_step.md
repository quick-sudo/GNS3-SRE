*******
Лабораторная работа №2: Мониторинг сетевой инфраструктуры
*******
Цель работы: Настроить сбор метрик сетевого оборудования с помощью стека мониторинга на основе SNMP, Prometheus и Grafana.

Используемый стек: Docker, Prometheus, SNMP Expoerter и Grafana.

В данной работе представлена следующая топология сети:

<img width="1047" height="770" alt="Image" src="https://github.com/user-attachments/assets/39eb8b66-b896-45bd-b315-a4bed01d1418" />

Где R1, R2 и R3 - роутеры Cisco 3725. Интрефейсы fa 0/0 имеют IP адреса 192.168.50.51, 192.168.50.52, 192.168.50.53 соответственно. Интрефейсы fa 0/1 имеют IP адреса 10.0.0.1, 15.0.0.1, 20.0.0.1 соответственно. VPCS - легковесный симулятор ПК от GNS3, поддерживающий протоколы DHCP, ICMP.

Для выполнения работы требуется:

1) Для обеспечения сетевой связности между всеми устройствами необхомино настроить OSPF. Для этого требуется запустить плейбук ospf_setup.yaml
2) Требуется настроить SNMP на самих роутерах. Проще всего это делается через SSH:
```cli
Router# configure terminal
Router(config)# snmp-server community public RO  
Router(config)# snmp-server host 192.168.50.10 version 2c public
exit
```
3) Необходимо развернуть сбор метрик при помощи плейбука deploy_snmp.yaml
4) В браузере необходимо открыть localhost:3000, добавить Data sources, и добавить Dashboard с ID 11159.
5) В конечном итоге должна быть открыта страница с метриками роутеров:
